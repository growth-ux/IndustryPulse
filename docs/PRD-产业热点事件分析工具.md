# 产品需求文档 (PRD)

## 产业热点事件爬取与时间脉络发展分析工具

---

## 1. 产品概述

### 1.1 产品定位

**产品名称**：IndustryPulse（产业脉搏）

**一句话描述**：输入一个产业关键词，自动生成该领域热点事件的时间轴，并配以AI生成的影响分析，帮助用户快速掌握产业发展脉络。

**核心价值主张**：
- **效率**：将数天的信息收集工作压缩到几分钟
- **洞察**：AI自动提炼事件影响，无需人工阅读原文
- **脉络**：时间轴形式呈现，清晰展示事件演进逻辑

### 1.2 目标用户

| 用户画像 | 场景 | 核心诉求 |
|---------|------|----------|
| 产业分析师 | 撰写研究报告 | 快速获取产业事件摘要 |
| 投资经理 | 尽职调查/投后管理 | 追踪被投企业行业动态 |
| 企业战略部 | 竞品分析/市场洞察 | 了解行业趋势变化 |
| 记者/编辑 | 选题策划 | 掌握行业热点线索 |
| 学术研究者 | 文献综述 | 梳理技术发展历程 |

### 1.3 名词定义

| 术语 | 定义 |
|------|------|
| 产业关键词 | 用户输入的分析主体，如"人工智能"、"新能源汽车" |
| 热点事件 | 一定时间内（周/月/季度/半年/年）该产业的重要新闻 |
| AI点评 | 系统自动生成的50字以内事件影响分析 |
| 事件类型 | 热点事件的分类维度（政策/融资/产品发布/并购等） |
| 时间轴 | 按时间顺序排列的事件展示形式 |

## 2. 功能需求

### 2.1 核心功能清单

#### F1: 关键词输入
- 单关键词输入（暂不支持多关键词组合）
- 支持中英文关键词
- 预设热门产业标签供快速选择：
  - 人工智能、新能源汽车、半导体、医疗健康、金融科技、教育科技、元宇宙、Web3

#### F2: 时间范围选择
- 预设选项：近一周、近一月、近一季、近半年、近一年
- 默认选项：近一月
- 自定义时间范围（开始日期 - 结束日期）

#### F3: 事件时间轴展示
- 垂直时间轴布局，左侧时间节点，右侧事件卡片
- 时间轴支持缩放：全部视图 / 按月分组
- 事件卡片默认收起，点击展开详情
- 滚动加载更多事件（分页/无限滚动）

#### F4: 事件卡片内容
每个事件卡片包含：
- 日期（精确到日）
- 事件标题（原文标题，可点击跳转原文）
- 来源媒体（带媒体logo图标）
- 摘要（原文前100字，AI截取核心句）
- AI点评（50字以内影响分析）
- 事件类型标签

#### F5: 事件类型分类
| 类型 | 说明 | 示例 |
|------|------|------|
| 政策 | 政府监管、扶持政策 | "工信部发布AI发展规划" |
| 融资 | 投融资事件 | "商汤科技完成新一轮融资" |
| 产品发布 | 技术/产品重要更新 | "OpenAI发布GPT-5" |
| 并购 | 收购、合并、战略合作 | "微软收购某AI创业公司" |
| 技术突破 | 重大技术进展 | "某团队实现芯片性能突破" |
| 财报 | 业绩披露 | "某巨头Q3财报营收增长" |
| 财报 | 业绩披露 | "某巨头Q3财报营收增长" |
| 人物 | 高管变动、观点发表 | "某CEO发表AI伦理观点" |
| 其他 | 不属于以上类型 |  |

#### F6: 事件筛选与搜索
- 按事件类型多选筛选
- 关键词高亮显示（搜索结果中标注匹配词）

#### F7: 数据导出
- 导出为JSON格式（包含完整数据结构）
- 导出为Markdown格式（便于粘贴到文档）

## 3. 数据需求

### 3.1 数据来源

| 媒体 | RSS链接 | 覆盖领域 | 优先级 |
|------|---------|----------|--------|

| 虎嗅 | https://www.huxiu.com/rss/ | 商业、科技 | 高 |

| 平台 | 数据类型 | 获取方式 |
|------|----------|----------|

| 同花顺 | 财经新闻 | API |

| 用途 | 工具 | 说明 |

## 5. UI/UX 设计方向（待定）

### 5.1 设计风格

**方向A：数据仪表盘风格**
- 深色主题，数据可视化优先
- 适合专业分析师

**方向B：简洁阅读风格**
- 浅色主题，类似新闻阅读器
- 适合快速浏览

**方向C：信息图风格**
- 时间轴用大字号、视觉化呈现
- 适合演示场景

### 5.2 推荐方案

采用**方向A+方向B结合**：
- 整体浅色主题，保证阅读舒适度
- 数据可视化元素（时间轴、标签）用深色/彩色点缀
- 参考Bloomberg Terminal的信息密度 + Medium的阅读体验

## 6. 非功能需求

### 6.1 可用性

- 支持Chrome、Safari、Firefox、Edge最新版
- 移动端基本可用（响应式布局）
- 无障碍支持：键盘导航、屏幕阅读器兼容

### 6.2 数据合规

- 仅采集公开可访问的新闻内容
- 注明数据来源，不声称内容版权
- 遵循robots.txt协议

### 6.3 错误处理

| 错误场景 | 用户提示 | 降级策略 |
|----------|----------|----------|
| 关键词无结果 | "未找到相关事件，尝试其他关键词" | 显示空状态 |
| 关键词过于宽泛 | "找到过多结果（>1000条），请缩小范围" | 自动建议筛选 |
| AI服务超时 | "AI分析超时，显示原始摘要" | 仅显示摘要，跳过点评 |
| 爬取失败 | "部分新闻源暂时不可用" | 显示可用数据，标注来源 |

## 9. 风险与挑战

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| RSS源被封禁 | 中 | 高 | 多源备份、及时切换 |
| AI成本超预期 | 高 | 中 | 缓存复用、分级处理 |

| 事件分类不准确 | 高 | 中 | 人工兜底、提供反馈入口 |
| 爬虫维护成本高 | 中 | 高 | 优先使用稳定RSS源 |
