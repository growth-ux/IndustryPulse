方案 B：工程化增强架构

FastAPI + React + MySQL + Redis + Celery，强调“真实系统设计”和复试展示完整度。

系统组件关系
React/Vite 前端
关键词输入、时间范围、时间轴、右侧洞察、手动刷新按钮
FastAPI 后端
查询 API、任务触发 API、分类/点评服务封装
MySQL
事件、来源、抓取任务、分类结果、AI 点评
Celery Worker
RSS 抓取、解析、去重、分类、点评、入库
Redis
任务队列 Broker / 任务状态
Celery Beat
定时预抓取，按 RSS 源周期调度
查询链路：React → FastAPI → MySQL → 返回时间轴与洞察
抓取链路：Celery Beat/手动按钮 → Redis → Celery Worker → RSS → MySQL
